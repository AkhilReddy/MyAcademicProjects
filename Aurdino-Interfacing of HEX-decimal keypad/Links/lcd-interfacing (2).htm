Displaying 5x10 font Character on lcd

10/1/2014

0 Comments

 
Character lcds can display character in 5x8 or 5x10 size dot-matrix. IN 5xN, 5 represents number of Coulombs and N represents number of Rows. Character Lcd's especially which are controlled by Hitachi HD44780 Controller can display 5x8 and 5x10 size font character. Some Lcd's can only display character in 5x8 font. In this Project We are going to display characters on lcd in 5x8 and 5x10 Dot-Matrix. 

We have to initialize Lcd (8x1, 8x2, 8x4, 16x1, 16x2, 16x4, 20x1, 20x2, 20x4, 24x1, 24x2, 24x4, 32x1, 32x2, 40x1, 40x2, 40x4  ) before using it. By initialization I mean 
  • Should the Cursor Appears on Lcd or not. If appearing, whether it should be Blinking or normal.
  • What Should be the Size of Character-font appearing on lcd (5x8, 5x10).

To learn about Lcd initialization Standard Commands go through the simple tutorial. 
COMMANDS INITIALIZING CHARACTER LCD

We are going to discuss only commands that are related to Character Size Font-matrix in this tutorial. To intialize character lcd we have to send commands to command-register of lcd. All the commands are 8-bit in nature. For Intializing  Character Font size, 8-bit commands individual bits represents
Character Lcd Font Size Setting command individual bits.
  • DL selects lcd mode 8-bit or 4-bit Dont know Go through the tutorial(LCD in 4-BIT and 8-BIT MODE )
  • N selects if lcd is of 2 rows or 1
  • F selects Character Font Size

Now if I have a 16x2 lcd and i want to initialize its Character font to be 5x10 my Command will be
Initializing 16x2 lcd to Display font in 5x10 size matrix.
Standard Lcd Commnads for 5x10 Display.
Lcd commands for 5x10 Size character Display.
  • You can display 208 characters in 5x8 Font and 32 characters in 5x10 font. Total ASCII characters present in HD44780 are 240. In which 208 are in 5x8 font and 32 are in 5x10 font. 
  • In CG-RAM You can create 8 custom characters at a time in 5x8 font and 4 custom characters in 5x10 font. Don't know about CG-RAM and Custom Characters Go through(CG-RAM & CUSTOM CHARACTER GENERATION). 

Now i am going to Display charcters on lcd in 5x8 and 5x10 font. I am going to print it on lcd using two different microcontrollers PIC and 8051(89c51,89c52). 

Displaying 5x10, 5x8 Characters on character lcd Using PIC 16f877 Microcontroller.

                      Project Requirements
  • PIC 16f877 microcontroller
  • 16x2 lcd
  • Potentiometer (To set Lcd Contrast)
  • Crystal (20 MHz)
  • Capacitors 33Pf

I am going to print G,J,P & Q in 5x8 and 5x10 font. These Characters are present in HD44780 Controller in 5x8 and 5x10 font. I will call their adresses and they will appear on lcd. See their Presence in the table given in the DATASHEET FOR HD44780 Controller. 
5x8 and 5x10 characters in lcd.
Lcd data pins are Connected to Port-B of Microcontroller. Lcd Control Pins (en,rs,rw) are connected to Port-D Pins#7,6,5. Rest of the connections are normal Connections. Suppling Power to Microcontroller. Grounding GND pin. Connecting Crystal to Microcontroller etc. Circuit Diagram of the Project is Given Below.
Lcd with Pic 16f877 microcontroller---Circuit Diagram

#include <htc.h>
#define _XTAL_FREQ 20000000
#define en RD7
#define rs RD6
#define rw RD5

void delay(unsigned int time)  //Time delay function
{
unsigned int i,j;
for(i=0;i<time;i++)
for(j=0;j<3;j++);
}

             //Function for sending values to the command register of LCD
void lcdcmd(unsigned char value)  
{
PORTB=value;
rs= 0;           //register select-rs 
rw = 0;      //read-write-rd
en = 1;    //enable-e
delay(50);
en=0;       //enable-e
delay(50);

}
             //Function for sending values to the data register of LCD
void display(unsigned char value)  
{
PORTB=value;
rs= 1;       //register select-rs 
rw= 0;       //read-write-rd
en= 1;       //enable-e
delay(500);
en=0;        //enable-e
delay(50);

}
             //function to initialize the registers and pins of LCD
             //always use with every lcd of hitachi
void lcdint(void)         
{
TRISB=0x00;    //Port B is used as output port             
TRISD5=0;
TRISD6=0;
TRISD7=0; 
     delay(15000);
     display(0x30);
     delay(4500);
     display(0x30);
     delay(300);
     display(0x30);
     delay(650);
lcdcmd(0x3C);  //5x10 Font Selected
delay(50);
lcdcmd(0x0C);  //Display on Cursor off 
delay(50);
lcdcmd(0x01);  //Clear Lcd 
delay(50);
lcdcmd(0x06);  //Entry Mode, Increment Cursor Automatically 
delay(50);
lcdcmd(0x80);  //Control set at Lcd 1 Row, 1 Coulomb  
delay(50);
}



void main()
{
lcdint();
display(0xea);//5x10 J
display(0x6a);//5x8 j
display(' ');
display(0x67);//5x8 g
display(0xe7);//5x10 G

display(0x70);//5x8 p
display(0xf0);//5x10 P
display(' ');
display(0x71);//5x8 q
display(0xf1);//5x10 Q
while(1);
}

Custom Characters with Pic 16f877

0 Comments

 
Here is a simple project on how to build custom characters in 16x2 lcd and then print them on lcd. Building and displaying self made custom characters on lcd is not a very hard task. To carry out this task you must know the internal structure of lcd. The size of the Ram, Registers of the lcd, CG-RAM(Character generated ram) of lcd. This CG-RAM is the most important part of lcd for generating and displaying self made characters. CG-RAM is fully disscused in the tutorials below. I recommend you to please take the tutorials other wise you will be unable to understand the code below.

  • How to generate Custom Characters in CG-RAM and display them on lcd.
  • 16x2 lcd working, pinout & registers.

Project Requirements.
  • Microcontroller PIC 16f877
  • 16x2 lcd
  • Potentiometer (For Setting lcd Contrast)
  • Crystal 20MHz
  • Connecting wires, Power Supply, Bread board or PCB for circuit.

Circuit diagram of the project is given below. Port-B of PIC 16f877 is connected with data pins of lcd. LCD controlling pins en(enable), rs(register select), r/w(read/write) are connected to Port-D pin 6 and 7. R/W(Read/Write) pin is grounded because we are only writting to lcd, Grounding the pin enables the write operation for lcd. All the other connections are necessary connections to brought the Microcontroller and whole circuit in working condition.
Circuit Diagram
Code is written in c language using MP-LAB ide and High TECH C compiler is used to compile the code. First i included the header file htc.h. This file must be included in every project which is going to be compiled in High Tech c compiler. Then frequency of the crystal is defined which is 20 MHz. Lcd control pins are defined next. Then some character arrays are defined. These character arrays are actually the custom characters which we are going to display on lcd. Delay function is used to give some arbitrary delay where needed. lcdcmd() function is sending commands to lcd. display() funtion is displaying characters on lcd. lcdint() function is initializing our lcd. In the main function I am generating and then displaying character on lcd.

To understand the code you must first know the internal structure of lcd. The tutorials links given above are very helpful for understanding the working and internal structure of CG-RAM of lcd. If you didn't take the tutorials. I recommend you to please go through them before going through the code below.

Urdu Characters on lcd

7/21/2014

2 Comments

 
In this project i am going to display Urdu characters on 16x2 lcd using Pic 16f877 Microcontroller. Lcd HD4478 controller only contains Ascii character set in it and we can only display numeric and English alphabets directly on lcd screen because ascii character set includes numeric and standard english characters. Displaying self made custom characters on lcd requires knowledge about internal structure of lcd. To display Custom characters like urdu you should know about Lcd CG-RAM,Data & Command registers.

Project Requirements
  • Pic 16f877 microcontroller
  • 16x2 lcd
  • Crystal 20 MHz
  • Potentiometer (For setting lcd Contrast)

A good and perfect tutorial to understand the internal structure of lcd is given below. Since its very important and lengthy topic so it is not possible to explain it here. 
HOW TO GENERATE CUSTOM CHARACTERS ON LCD.
I recommend you to please take the above tutorial other wise you will be unable to understand the code below. 

Lcd is interfaced with Pic 16f877 microcontroller in 8-bit mode. Lcd data Pins are directly connected to Port-B of Pic 16f877  microcontroller. Lcd control signals rw(read-write) rs(register select) and en(enable) are provided by Port-D pins RD5,RD6 and RD7 of Pic 16f877 microcontroler. The circuit diagram of the project is given below.
Urdu Alphabets on 16x2 lcd -- Circuit Diagram
Coming to the code portion. The code is written in c language using MP-lab ide. High tech c compiler is used to compile the code. Header file htc.h is imported first. This file is to included in every project that is using high tech c compiler for compiling the code. Then crystal frequency is defined which is 20 MHz(20e6=20^6). Lcd control pins are defined next. Then the delay() function is defined which is generating some delay for us. lcdcmd() function is sending commands to command register of lcd. dispaly() function is sending data to data register of lcd. If you Dont Know about lcd data and command register and their working here is a good tutorial.

LCD DATA AND COMMAND REGISTERS WORKING.

Lcdint() function is initializing our lcd. The tutorial given a line above also covers the lcd intialization phase and intializing commands. In the main fuction i am generating urdu alphabets in CG-RAM and then diaplay them on 16x2lcd. First i give command lcdcmd(0x40) to lcd. This 0x40 is the starting address of CG-RAM. Then the individual dots of urdu first character "ALLIF" is send to lcd. Next 0x48 is send to lcd. Which is the second address of CG-RAM Character. Second Character dots are then send to CG-RAM of lcd. This Process repeats for the eight characters. Then all the eight Custom generated characters are invoked for displaying on lcd by just giving their address(0 to 8).

After Displaying first eight urdu characters next eight are build in CG-RAM and then displayed on lcd. This process repeats until the last character is displayed. After last character the lcd starts from beginning and starts displaying urdu characters from beginning. 

Note: Lcd first line continuously display "Urdu Characters!" and the urdu characters are displayed on the second line. Each urdu character is displayed on lcd with one matrix gap from other. Since 16x2 lcd has 16 coulombs so 8 characters with one matrix sepration from others are perfectly displayed on the lcd second line.

Ascii Characters on 16x2 lcd Using PIC 16f877 Microcontroller

7/8/2014

0 Comments

 
In this tutorial i am going to print ascii characters on 16x2 lcd using PIC 16f877 microcontroller. Lcd is interfaced with Pic microcontroller in 8-bit mode. The code is written in c language using high tech c compiler and Mp-lab. Lcd interfacing with microcontroller and code is very easy. Only the portion which is little bit complex is how generate ascii characters. Well you dont need to generate ascii characters they are already present in the lcd controller (HD44780). You just need to know how to invoke the ascii characters to be displayed on the screen. In the code portion it is explained.

Project Requirements
  • Pic 16f877 microcontroller.
  • 16x2 lcd. 
  • Crystal 20MHz
  • Power Supply
  • Bread board or Design PCB(Printed circuit board)
  • Potentiometer (For setting lcd contrast)

If you are new in the field of microcontrollers and lcds and dont know about lcd pinout,working and internal structure of lcd then please go through the following tutorials. They will clear you about the working of lcd data and command registers. It will also explain you about how to use lcd in 4-bit and 8-bit mode. 
  • 16x2 LCD PINOUT AND WORKING.

The circuit diagram of the project is given below. Port-B of Pic 16f877 is connected with data pins of 16x2 lcd. Which means Port-B is used to send commands and data to 16x2 lcd. Lcd controll signals are provided using individual bits of Port-D. All the other connections are normal connections appling vcc to microcontroller and lcd where vcc is necessary. Making GND pins ground. You can see the circuit diagram given below. 
Circuit Diagram
In the code process first include the header file htc.h. If you are using high tech c compiler always include this library. Then frequency of the oscillator is defined which is 20MHz. Then individual pins of Port-D are defined. These pins are used to provide controll signals to lcd. delay() function is to generate some delay where necessary. lcdcmd() function is sending commands to lcd with control signals. display() function is sending data to lcd with control signals.lcdint() function is initializing our lcd(4-bit mode,8-bit mode, display on ,cursor off etc). 

In the main function two instructions are invoking ascii characters. The instructions i=j/10; where j is integer(int) and i is character(char). Now when we divide two integers and save result in character(char) variable. The result is stored in ascii format. Since j is 0 and dividing 0 by 10 gives 0. s0 i contains 0 or ascii value of zero.

The ascii character 0 is present at address 0x30. To go to address 0x00 negate 0x30 from 0x30.The instruction i=i-0x30; is doing the same job. First i contains 0x30 after executing i=i-0x30 i contains 0x00. Hence we are at starting address of ascii characters. Now increment the address one by one and display the ascii character associated with that address on the lcd screen.  

Note that the ascii characters of the HD44780 controller differs from the standard ascii characters. The HD44780 controller contains ascii characters in the following format. Some addresses are also void. So dont get confused when you see the characters like below displayed on your lcd screen.
Ascii Characters of HD44780 Controller
#include <htc.h>
#define _XTAL_FREQ 20000000
#define en RD7
#define rs RD6
#define rw RD5

void delay(unsigned int time)  //Time delay function
{
unsigned int i,j;
for(i=0;i<time;i++)
for(j=0;j<5;j++);
}

             //Function for sending values to the command register of LCD
void lcdcmd(unsigned char value)  
{
PORTB=value;
rs= 0;           //register select-rs 
rw = 0;      //read-write-rd
en = 1;    //enable-e
delay(50);
en=0;       //enable-e
delay(50);

}
             //Function for sending values to the data register of LCD
void display(unsigned char value)  
{
PORTB=value;
rs= 1;       //register select-rs 
rw= 0;       //read-write-rd
en= 1;       //enable-e
delay(500);
en=0;        //enable-e
delay(50);

}
             //function to initialize the registers and pins of LCD
             //always use with every lcd of hitachi
void lcdint(void)         
{
TRISB=0x00;    //Port 1 is used as output port             
TRISD5=0;
TRISD6=0;
TRISD7=0; 
    delay(15000);
     display(0x30);
     delay(4500);
     display(0x30);
     delay(300);
     display(0x30);
     delay(650);

delay(50);

delay(50);
lcdcmd(0x01);
delay(50);
lcdcmd(0x06);
delay(50);
lcdcmd(0x80);
delay(50);
}

j=0;k=0;

i=i-0x30;
lcdint();

display(i);j++;i++;k++;

if(k==15){
lcdcmd(0xC0);

lcdcmd(0x80);
